Introduction & Motivation

The Rise of Quantum Computing (QC)

  • QC leverages quantum mechanics for powerful computation.
  • Potential to solve problems intractable for classical computers (e.g., drug discovery, cryptography).
  • However, QC is in its early stages and faces significant challenges:
    • Noise and high error rates
    • High cost of execution
    • Limited access to hardware

The Problem: A Gap in Quantum Tooling

Pre-runtime vs. Post-runtime Metrics

Pre-runtime Metrics

  • Calculated before execution (static analysis).
  • Examples: Number of qubits, circuit depth, gate counts.
  • Helpful for estimating resource requirements.

Post-runtime Metrics

  • Calculated after execution on a simulator or QPU.
  • Examples: Success rate, fidelity, execution time.
  • Crucial for understanding real-world performance.

Example: Understanding Metrics with a Bell State

The Bell State Circuit

Creates an entangled state: (|00⟩ + |11⟩) / √2

Key Metrics

Pre-runtime: Circuit Depth

Measures the number of sequential gate layers. Lower depth = less time for errors to accumulate.

Post-runtime: Success Rate

Percentage of measurements matching expected outcomes (|00⟩ or |11⟩).

Post-runtime: Fidelity

Measures how close the actual quantum state is to the ideal theoretical state.

The Problem: Our Research Questions

  1. To what extent do existing QC SDKs facilitate the collection of pre- and the calculation of post-runtime metrics?
  2. Do QC SDKs effectively incorporate emerging metrics as reported in the literature?
  3. How can a library be designed to address the need to collect and analyze a wide spectrum of quantum circuit metrics?

Answering Our Questions

RQ1: Do existing SDKs facilitate metric collection?

Answer: Insufficiently. They offer only basic support.

  • Current SDKs provide fundamental characteristics like circuit depth, width, and gate counts.
  • However, they lack the advanced metrics extensively covered in academic literature.
  • There is a shortage of robust, built-in tools for detailed execution analysis.

Answering Our Questions

RQ2: Do SDKs incorporate emerging metrics from literature?

Answer: Evidently, no.

  • There is a clear disparity between theoretical advances in metrics and their practical implementation in SDKs.
  • This gap forces researchers and developers to repeatedly code the same analysis from scratch.
  • This leads to inconsistent metric definitions and varying calculation methodologies across projects.

Answering Our Questions

RQ3: How can a library address these gaps?

Answer: By providing a structured and extensible solution.

  • Implement the Strategy Pattern for consistent and easy integration of new metrics.
  • Use Schema Validation (e.g., Pydantic) to ensure data integrity and type safety.
  • Include Integrated Visualization Tools to translate raw data into actionable insights.
  • This is the philosophy behind qWard.

qWard in Action: A Practical Example

Live Demo Backup: Code Snippets

1. Create the circuit:

circuit = QuantumCircuit(2, 2)
circuit.h(0)
circuit.cx(0, 1)
circuit.measure([0,1], [0,1])

2. Integrate with qWard:

scanner = Scanner(
    circuit=circuit,
    job=jobs,
    strategies=[
        QiskitMetrics,
        ComplexityMetrics,
        CircuitPerformanceMetrics(...)
    ]
)
metrics_dict = scanner.calculate_metrics()

3. Visualize the results:

visualizer = Visualizer(scanner=scanner)
visualizer.generate_plots(
    selections={
        Metrics.CIRCUIT_PERFORMANCE: [
            Plots.CircuitPerformance.SUCCESS_ERROR_COMPARISON,
            Plots.CircuitPerformance.FIDELITY_COMPARISON
        ]
    }
)

Results: Post-runtime Metrics - Fidelity

How Close Are We to the Ideal Quantum State?

What is Fidelity?

Fidelity measures how similar the actual quantum state produced by the circuit is to the theoretical ideal state.

  • Fidelity = 1.0: Perfect match with ideal state
  • Fidelity < 1.0: Deviations due to noise, decoherence, and gate errors
  • This metric helps evaluate circuit optimizations and hardware quality
