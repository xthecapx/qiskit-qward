How Quantum Computing Works

1. Code
Write a quantum algorithm using gates and qubits
→
2. Transpile
Translate the circuit into the hardware's native gates
→
3. Execute
Run on a quantum processor many times (e.g., 1,024 shots)
→
4. Output
A histogram of results — which outcome appeared how often?
Research question: Current hardware is noisy — the output histogram often contains errors. How do we know if the quantum program actually worked? We propose the DSR (Differential Success Rate), a score that measures how clearly the expected answer stands out above the noise in the histogram.

DSR: Identifying the Components

The DSR measures the gap between the expected outcome and the strongest competitor in the histogram, normalized into a score from 0 to 1.

Dependent variable: DSR computed from the target state and the returned histogram  |  Additional metrics: Hellinger Fidelity (HF) and Total Variation Distance Fidelity (TVDF)

Statistical Analysis & Limitations

Non-parametric Statistical Tools
Wilcoxon signed-rank Tests whether paired scores (e.g., DSR vs. HF) differ systematically; $p$-values corrected with Bonferroni ($p_{\text{corr}} = p_{\text{raw}} \times 2$)
Cliff's $\delta$ Effect size: proportion of pairwise comparisons where one metric exceeds the other
Bootstrap 95% CI 10,000 resamples for median estimation (groups with $n \geq 5$)
Limitations
  • DSR requires known expected outcomes — not applicable to variational algorithms
  • Circuit depth alone is not a good predictor of DSR
  • Correctness of target states $E$ is essential
  • Shot noise and calibration drift affect values

DSR Degradation Simulation

As hardware noise increases, the expected outcome loses distinction in the histogram and the DSR score drops toward zero.
